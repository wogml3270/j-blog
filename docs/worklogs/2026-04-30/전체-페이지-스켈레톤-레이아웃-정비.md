# 전체 페이지 스켈레톤 레이아웃 정비

## 배경
- 공개/관리자 페이지의 `loading.tsx` 스켈레톤이 실제 레이아웃과 비율이 맞지 않아 화면 점프가 발생함.
- 모바일/데스크톱에서 툴바, 리스트, 상세 본문 배치가 서로 달라 체감 품질이 떨어짐.

## 변경 사항
- 공개 목록 공통 스켈레톤(`ContentListLoadingSkeleton`)을 검색 입력/버튼/정렬 + 카드 메타 구조에 맞게 재구성.
- 홈 히어로 스켈레톤(`ImmersiveHeroSkeleton`)을 실제 히어로 텍스트 위치/썸네일 레이어와 동일한 정렬로 보정.
- 블로그 상세 `loading.tsx`를 본문 + TOC(xl) + 태그 + 댓글 영역까지 포함하도록 확장.
- 프로젝트 상세 `loading.tsx`를 상단 메타/썸네일/리뷰/TOC/댓글 구조와 동일하게 확장.
- 관리자 공통 스켈레톤(`protected-loading-skeleton.tsx`)을 헤더/툴바/섹션 카드/페이지네이션 구조로 전면 정비.
- 관리자 탭별 로딩 props 보정:
  - home: 단일 섹션, 생성 버튼 없음
  - blog/projects: 2섹션
  - contact: 2섹션 + 썸네일 없는 행
- 공개 About 페이지 전용 로딩 추가:
  - `components/about/interactive-about-loading-skeleton.tsx`
  - `app/[lang]/(site)/about/loading.tsx`

## 검증
- `npx tsc --noEmit` 통과
- `npm run lint` 통과
- `npm run build`는 Supabase fetch 실패로 `/ko` prerender 단계에서 중단됨(네트워크/외부 의존 이슈)
