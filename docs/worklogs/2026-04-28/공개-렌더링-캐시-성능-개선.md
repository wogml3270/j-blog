# 공개 렌더링 캐시 성능 개선

## 배경
- 공개 사이트 체감 속도가 느리고 스켈레톤 노출 시간이 길다는 이슈가 발생했다.
- 원인 조사 결과, 공개 라우트가 강제로 request-time 동적 렌더링되고 있었고, Supabase 서비스 fetch가 전역 `no-store`로 고정되어 있었다.

## 원인
1. `app/[lang]/(site)/layout.tsx`에서 `dynamic = "force-dynamic"` + `headers()` 호출로 공개 라우트 전체가 동적 렌더링 강제됨.
2. `app/page.tsx`도 `force-dynamic`으로 설정되어 루트 경로 정적 최적화 이점이 사라짐.
3. `lib/supabase/service.ts`에서 모든 조회가 `cache: "no-store"`, `revalidate: 0`으로 강제되어 매 요청마다 DB 왕복 발생.
4. `lib/supabase/proxy.ts`에서 비로그인 요청까지 매번 `auth.getUser()`를 수행해 추가 지연 발생.

## 적용한 변경
- `app/[lang]/(site)/layout.tsx`
  - `force-dynamic` 제거
  - `headers()` 호출 제거
- `app/page.tsx`
  - `force-dynamic` 제거
- `lib/supabase/service.ts`
  - 서비스 클라이언트 캐시 전략 분리
  - `cacheStrategy: "no-store" | "revalidate"` 옵션 추가
  - 공개 조회용 ISR fetch(`force-cache + revalidate`) 지원
- 공개 조회 함수만 ISR 사용
  - `lib/blog/repository.ts` (`requireBlogService`)
  - `lib/projects/repository.ts` (`requireProjectService`)
  - `lib/profile/repository.ts` (`getPublishedAboutContent`)
  - `lib/home/repository.ts` (`getHomeSlides`)
- `lib/supabase/proxy.ts`
  - Supabase auth 쿠키가 없는 요청은 `auth.getUser()` 생략

## 검증
- `npx tsc --noEmit` 통과
- `npm run -s build` 통과
- 빌드 결과에서 `/[lang]`, `/[lang]/about` 경로가 `Revalidate 1m`로 생성되는 것 확인

## 기대 효과
- 공개 페이지 TTFB 개선
- 최초 진입 시 스켈레톤 체류 시간 감소
- 비로그인 사용자의 불필요한 auth 네트워크 왕복 제거
- 관리자/쓰기 경로는 기존 no-store 정책 유지로 데이터 일관성 보장

## 2차 보강: 페이지 이동 시 이미지 재요청 체감 완화
- 홈 히어로와 소개 페이지의 대형 이미지 렌더를 `<img>`에서 `next/image`로 전환.
- 아바타 이미지도 `next/image`로 통일해 브라우저/최적화 캐시 경로를 일관화.
- `next.config.ts`에 `images.minimumCacheTTL = 86400` 추가(최적화 이미지 캐시 TTL 1일).

### 수정 파일
- `components/home/immersive-hero-slider.tsx`
- `components/about/interactive-about-reveal.tsx`
- `components/layout/header.tsx`
- `next.config.ts`

## 3차 보강: 동적 라우트 재진입 시 스켈레톤 빈도 완화
- `next.config.ts`에 `experimental.staleTimes` 설정 추가.
  - `dynamic: 30`
  - `static: 300`
- 효과: `/blog`, `/projects` 같은 동적 페이지를 짧은 시간 안에 재방문할 때 라우터가 캐시된 RSC payload를 재사용해 loading 스켈레톤 노출 빈도를 줄인다.

## 4차 보강: About 기술스택 로고 재요청 완화
- About 기술스택 로고는 SVG 비중이 높아 원본 스토리지 헤더 영향을 직접 받는 경로였다.
- `/api/media-cache` 프록시 라우트를 추가해 Supabase public 이미지 응답을 앱 경유로 캐시(`max-age=31536000, immutable`) 처리.
- `interactive-about-reveal`의 기술스택 로고 src를 캐시 프록시 URL로 변경.
- 프로필 대형 이미지는 기존 `next/image` 유지, 기술스택 로고는 경량 렌더 안정성을 위해 `img` 태그로 유지.
